1 Nakama Unity
2 ============
3 
4 > Unity client for Nakama.
5 
6 Nakama is an open-source distributed server for social and realtime games. For more information have a look at the [server documentation](https://heroiclabs.com/docs/); and for a quick list of build targets run `gradle tasks`.
7 
8 The [client guide](https://heroiclabs.com/docs/clients/unity/) is available on the server documentation. The [client reference documentation](http://heroiclabs.github.io/nakama-unity/) is deployed on gh-pages.
9 
10 If you encounter any issues with the server you can generate diagnostics for us with `nakama doctor`. Send these to support@heroiclabs.com or [open an issue](https://github.com/heroiclabs/nakama/issues). If you experience any issues with the client, it can be useful to enable trace (`.Trace(true)`) to produce logs and [open an issue](https://github.com/heroiclabs/nakama-unity/issues).

33 ### Contribute
34 
35 To build the codebase you will need to install these dependencies:
36 
37 * __unity__ The Unity editor/compiler toolchain.
38 * __gradle__ A JVM-based build tool.
39 * __protobuf__ A toolchain used to create custom protocols.
40 * __doxygen__ A tool for generating documentation from annotated source.
41 
42 You can then install code dependencies:
43 
44 ```
45 $> gradle nugetRestore
46 ```
47 
48 For development use the compile and test run cycle:
49 
50 ```
51 $> gradle nunit
52 ```
53 
54 If needed you can build and run tests which use the UnityTestTools integration with the Unity editor. These can be run headless with `gradle unityTest`.
55 
56 To package a release use:
57 
58 ```
59 $> gradle unityRelease
60 ```
